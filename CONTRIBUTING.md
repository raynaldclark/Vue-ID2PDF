# 贡献指南

感谢您对 Vue-ID2PDF 项目的关注！我们非常欢迎任何形式的贡献，包括但不限于：功能改进、缺陷修复、文档完善、使用问题反馈等。

## 开发环境设置

1. 克隆仓库：
   ```bash
   git clone https://github.com/Vue-ID2PDF/Vue-ID2PDF.git
   cd Vue-ID2PDF
   ```

2. 安装依赖：
   ```bash
   npm install
   # 或者
   pnpm install
   ```

3. 启动开发服务器：
   ```bash
   npm run dev
   # 或者
   pnpm dev
   ```

## 贡献流程

1. 在开始工作之前，请先查看 [Issues](https://github.com/Vue-ID2PDF/issues) 列表，确认您想解决的问题或添加的功能是否已有人在处理。

2. Fork 本仓库到您的 GitHub 账号下。

3. 基于 main 分支创建一个新的分支：
   ```bash
   git checkout -b feature/your-feature-name
   # 或者
   git checkout -b fix/issue-you-are-fixing
   ```

4. 在您的分支上进行开发和测试。

5. 提交您的更改，请使用有意义的提交信息：
   ```bash
   git commit -m "feat: add new feature"
   # 或者
   git commit -m "fix: resolve issue with camera detection"
   ```

6. 将您的分支推送到您的 GitHub 账号中的 fork：
   ```bash
   git push origin feature/your-feature-name
   ```

7. 在 GitHub 上创建一个新的 Pull Request，从您的分支合并到原仓库的 main 分支。

## 代码规范

- 请遵循项目既有的代码风格和结构
- 所有的代码更改应该包含适当的注释
- 确保您的代码没有 linting 错误
- 确保您的代码能在不同环境中正常运行（桌面端和移动端）

## 问题报告

如果您发现了 bug 但没有时间修复它，请创建一个 Issue，并尽可能提供以下信息：

- 问题的详细描述
- 复现步骤
- 预期的行为和实际行为
- 浏览器及操作系统版本
- 如可能，提供截图或录屏

## 功能请求

如果您希望添加新功能，请创建一个 Issue，并描述：

- 您希望添加的功能
- 为什么这个功能会对项目有益
- 您对实现的初步想法（如有）

## 文档改进

文档是项目的重要组成部分。如果您发现文档中的错误或有改进建议，欢迎提交更改。

## 许可

通过向本项目贡献代码，您同意您的贡献将根据本项目的 [MIT 许可](LICENSE) 进行许可。
